Finding Waldo using Grover Search#

The files for this tutorial may be found here: IBM-Quantum-Technical-Enablement/waldo-grovers

Background#

Grover’s algorithm is an unstructured search algorithm that aims to find the input to a black-box function/oracle that returns a desired output value with high-probability. Here we will show how to formulate such a search problem using Qiskit and it’s circuit library to find Waldo within the following image:

# Plotting tools
import matplotlib.pyplot as plt
from PIL import Image

fig, axes = plt.subplots(4, 4, figsize=[1.5*6.4, 1.5*4.8])
for idx in range(16):
    row_ind = idx // 4
    current_axes = axes[row_ind, idx - 4*row_ind]
    current_axes.imshow(Image.open('split/{}.jpg'.format(idx)))
    current_axes.axis('off')
plt.subplots_adjust(wspace=0, hspace=0)

The notion of “where is Waldo” needs to be made a bit more concrete before we can actually attempt such a search. Here we will descretize the image into a 4x4 grid, and finding Waldo will come down to which of these 16 sub-images he is in. Here the images are split in row-order as follows:

fig, axes = plt.subplots(4, 4, figsize=[1.5*6.4, 1.5*4.8])
for idx in range(16):
    row_ind = idx // 4
    current_axes = axes[row_ind, idx - 4*row_ind]
    current_axes.imshow(Image.open('split/{}.jpg'.format(idx)))
    current_axes.axis('off')
    current_axes.set_title('image #{}'.format(idx), fontsize=10)
plt.subplots_adjust(wspace=0.3, hspace=0.1)

Setup#

The imports needed to run this notebook. The plotting utilities are loaded above to display the images.

# Built-in modules
import math

# Imports from Qiskit
from qiskit import QuantumCircuit
from qiskit.circuit.library import GroverOperator, MCMT, ZGate
from qiskit.visualization import plot_distribution

# Imports from Qiskit Runtime
from qiskit_ibm_runtime import QiskitRuntimeService, Sampler

Load Qiskit Runtime#

Here we load the runtime service, and select our target backends; one simulator for verification, and a real-device.

service = QiskitRuntimeService()

sim = service.get_backend("ibmq_qasm_simulator")
backend = service.get_backend("ibm_cusco")

Oracle definition#

The oracle definition we use here is based on the CZ gate, which marks the all-ones bit-string with a negative phase value. This is exactly what we want for a Grover oracle. However, we are likely not interested in the all-ones bit-string, but rather some bit-string containing both ones and zeros. We can modify the CZ gate, or its multi-controlled version, to mark states with zeros by using X gates on the qubits with a zero to yield an open-control on those qubits.

def grover_oracle(marked_states):
    """Build a Grover oracle for multiple marked states

    Here we assume all input marked states have the same number of bits

    Parameters:
        marked_states (str or list): Marked states of oracle

    Returns:
        QuantumCircuit: Quantum circuit representing Grover oracle
    """
    if not isinstance(marked_states, list):
        marked_states = [marked_states]
    # Compute the number of qubits in circuit
    num_qubits = len(marked_states[0])

    qc = QuantumCircuit(num_qubits)
    # Mark each target state in the input list
    for target in marked_states:
        # Flip target bit-string to match Qiskit bit-ordering
        rev_target = target[::-1]
        # Find the indices of all the '0' elements in bit-string
        zero_inds = [
            ind for ind in range(num_qubits) if rev_target.startswith("0", ind)
        ]
        # Add a multi-controlled Z-gate with pre- and post-applied X-gates (open-controls)
        # where the target bit-string has a '0' entry
        qc.x(zero_inds)
        qc.compose(MCMT(ZGate(), num_qubits - 1, 1), inplace=True)
        qc.x(zero_inds)
    return qc

Setting marked state from images#

We require one or more “marked” states for Grover search. Here I have marked the state with Waldo (a bit hastily) in it in the EXIF image data. Below is a function that finds the marked state and returns the corresponding bit-string.

def get_marked_image():
    """Get the bit-string corresponding to the marked state with Waldo in it.
    
    Returns:
        str: Bit-string for marked image 
    """
    marked_image = None
    for idx in range(16):
        img = Image.open('split/{}.jpg'.format(idx))
        exif_data = img._getexif()
        if exif_data:
            if 34665 in exif_data:
                marked_image = bin(idx)[2:].zfill(4)
    if not marked_image:
        raise Exception('Image not found')
    return marked_image

Let us now use the above function to find the image with Waldo, and generate the corresponding Grover oracle:

marked_states = [get_marked_image()]
oracle = grover_oracle(marked_states)
oracle.draw("mpl")

Full Grover circuit#

To perform Grover iteration we need our oracle, and an operator that performs the amplification. This is provided in the circuit library via the GroverOperator. We can simply pass our oracle into this circuit building class:

grover_op = GroverOperator(oracle)
grover_op.decompose(reps=1).draw("mpl")

We now have a single iteration of our Grover search, but that might not be enough. Here we find the optimal number which is determined by the ratio of number of marked states to the total Hilbert space dimensionality:

optimal_num_iterations = math.floor(
    math.pi / 4 * math.sqrt(2**grover_op.num_qubits / len(marked_states))
)

The final circuit becomes a state preparation of an equal superposition over all states using Hadamard gates, followed by repeated applications of our Grover iteration circuit the optimal number of times:

qc = QuantumCircuit(grover_op.num_qubits)
# Create even superposition of all basis states
qc.h(range(grover_op.num_qubits))
# Apply Grover operator the optimal number of times
qc.compose(grover_op.power(optimal_num_iterations), inplace=True)
# Measure all qubits
qc.measure_all()
qc.draw("mpl")

Verification on classical simulator#

Because our circuit is small, \(\lesssim 30\) qubits, we can easily validate it one a classical simulator. Here we choose to sample our circuit using the cloud simulator:

sim_sampler = Sampler(sim)

sim_dist = sim_sampler.run(qc, shots=int(1e4)).result().quasi_dists[0]

We plot the resulting distribution whose integer of maximal probability is (hopefully) the image where Waldo is located.

plot_distribution(sim_dist)

Lets plot the sub-image to the corresponding image and see if we are correct:

fig, ax = plt.subplots()
ax.imshow(Image.open('split/{}.jpg'.format(max(sim_dist, key=sim_dist.get))))
ax.axis('off');

If everything worked, we should see Waldo on the far right.

Execution on a real system#

We can now repeat our experiment on a real system, and see if it too can find the correct image.

real_sampler = Sampler(backend)

real_dist = real_sampler.run(qc, shots=int(1e4)).result().quasi_dists[0]

fig, ax = plt.subplots()
ax.imshow(Image.open('split/{}.jpg'.format(max(real_dist, key=real_dist.get))))
ax.axis('off');

Indeed it does. However, the corresponding probability is a fair bit lower.

plot_distribution(real_dist)

Grover search is a problem very ill-suited for near-term hardware as the circuit depth grows quite rapidly with qubit count. We could probably increase the granularity of the image to 5x6, which fits a 5-qubit circuit, but anything beyond that is likely to give poor results unless we put in a fair bit of effort to try to suppress errors.
