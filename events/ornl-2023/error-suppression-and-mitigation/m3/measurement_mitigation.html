For measurement error mitigation, we will use the matrix-free measurement mitigation (mthree) package.

Python Package:
https://qiskit.org/ecosystem/mthree/index.html

Paper:
Scalable Mitigation of Measurement Errors on Quantum Computers
Paul D. Nation, Hwajung Kang, Neereja Sundaresan, and Jay M. Gambetta
PRX Quantum 2, 040326 – Published 8 November 2021
https://journals.aps.org/prxquantum/abstract/10.1103/PRXQuantum.2.040326

Setup#

def create_hadamard_chain(num_qubits):
    qc = QuantumCircuit(num_qubits)
    qc.x(range(0, num_qubits))
    qc.h(range(0, num_qubits))
    for kk in range(num_qubits//2,0,-1):
        qc.ch(kk, kk-1)
    for kk in range(num_qubits//2, N-1):
        qc.ch(kk, kk+1)
        
    return qc

N = 12
qc = create_hadamard_chain(num_qubits=N)
qc.draw('mpl', fold=-1)

qc_plus_meas = qc.measure_all(inplace=False)

backend = FakeMumbai()

ideal_simulator = AerSimulator()

trans_qc = transpile(qc_plus_meas, backend, optimization_level=3)

Getting final maps of physical qubits using mthree#

To mitigate measurement error, we need to know finally from which physical qubits the data was read out because depending on the physical qubits the measurement error rates can vary. With M3, finding out the final physical qubits (final measurement maps) are as easy as a function call. Note that, we need to supply the transpiled quantum circuits as they have the accurate final measurement mappings (Transpiler can map the virtual qubits of your circuit to any set of physical qubits depening on the choice of initial layout (map) method. Besides, the routing stage of the transpiler can change layout to meet the device connectivity. Thus, your final measurement mappings can be non-trivial).

measured_qubits = mthree.utils.final_measurement_mapping(trans_qc)

Run measurement calibration#

After knowing the final measurement mappings, we need to run calibration circuits to quanitfy the readout error probabilities.

# Calibration
mit = mthree.M3Mitigation(backend)
mit.cals_from_system(measured_qubits)

Executing main circuit#

raw_counts = backend.run(trans_qc, shots=10000).result().get_counts()

raw_expval = sampled_expectation_value(dist=raw_counts, oper="Z"*N)

Applying measurement mitigation#

# Applying measurement correction using M3, converts the Counts into quasi probability distributions
quasi_probs = mit.apply_correction(raw_counts, measured_qubits)

mitigated_expval = sampled_expectation_value(dist=quasi_probs, oper="Z"*N)

ideal_counts = ideal_simulator.run(trans_qc, shots=10000).result().get_counts()

ideal_expval = sampled_expectation_value(dist=ideal_counts, oper="Z"*N)

print(f"Ideal expectation value: {ideal_expval:0.4f}")
print(f"Raw expectation value: {raw_expval:0.4f}")
print(f"Mitigated expectation value: {mitigated_expval:0.4f}")
