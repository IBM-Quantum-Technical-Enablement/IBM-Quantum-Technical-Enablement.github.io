Imports#

import numpy as np
from qiskit import *
from qiskit.quantum_info import hellinger_fidelity

from qiskit_aer import AerSimulator
from qiskit_ibm_provider import IBMProvider

# Plotting tools
import matplotlib.pyplot as plt
import matplotlib_inline
matplotlib_inline.backend_inline.set_matplotlib_formats('retina')

Setup#

provider = IBMProvider()
backend = provider.get_backend("ibm_peekskill")
ideal_simulator = AerSimulator()
noisy_simulator = AerSimulator.from_backend(backend)

def bv_ones_circs(N):
    """Create a Bernstein-Vazirani circuit with all-ones oracle of width N
    """
    qc = QuantumCircuit(N, N-1)
    qc.x(N-1)
    qc.h(range(N))
    qc.cx(range(N-1), N-1)
    qc.h(range(N-1))
    qc.barrier()
    qc.measure(range(N-1), range(N-1))
    return qc

N = 5
qc = bv_ones_circs(N)
qc.draw('mpl')

Now, obviously we can run this circuit (after transpiling) on IBM Quantum hardware. However, this circuit has a couple of issues that limit the output fidelity. (1) The circuits requires \(N+1\) qubits to encode a length \(N\) bit-string. (2) On the current fleet of IBM Quantum systems based on the heavy-hex architecture, any algorithm requiring more than 3 CNOT gates per qubit needs to be re-routed to match the constraints of the device, adding additional swap gates.

Obviously, if we wanted to extend our BV circuits to larger numbers of qubits, then the additional swaps would greatly limit the resulting fidelity. We can see this easily with a noisy simulator and/or execution on hardware.

Typical Execution#

Now we will generate BV circuits for all-ones bit-string (worst case scenario) of length from \(2\) to \(12\)

MAX_BITS = 10
bv_circs = [bv_ones_circs(N) for N in range(2, MAX_BITS+1)]

trans_bv_circs = transpile(bv_circs, backend=backend, optimization_level=3)

ideal_counts = ideal_simulator.run(trans_bv_circs, shots=10000).result().get_counts()

noisy_sim_counts = noisy_simulator.run(trans_bv_circs, shots=10000).result().get_counts()

noisy_sim_fidelities = [hellinger_fidelity(noisy, ideal) for noisy, ideal in zip(noisy_sim_counts, ideal_counts)]

real_counts = backend.run(trans_bv_circs, shots=10000).result().get_counts()

real_fidelities = [hellinger_fidelity(real, ideal) for real, ideal in zip(real_counts, ideal_counts)]

fig, ax = plt.subplots()
ax.plot(range(2, MAX_BITS+1), noisy_sim_fidelities, "o--", ms=10, label='Simulation')
ax.plot(range(2, MAX_BITS+1), real_fidelities, "o--", ms=10, label='Hardware')
ax.set_xlabel('Bits')
ax.set_ylabel('Fidelity')
ax.legend();

Dynamic Circuits#

Up until now we have focused on circuits with a set pattern of instructions: gates, measurements, etc. It is a deterministic execution model.

As in classical computation, the richness of expression that is offered by flow-control in quantum circuits allows us to perform computations that are more efficient, or would otherwise be impossible.

Our first dynamic operation is the reset operation, that returns the qubit to the \(|0\rangle\) state:

qc_reset = QuantumCircuit(1)
qc_reset.reset(0)
qc_reset.draw('mpl')

Although it looks like any other operation, on IBM Quantum systems the reset operation is actually dynamic, consisting of a measurement operation followed by an x-gate conditioned on the (classical) value of the measurement.

This can be written in Qiskit in the following way:

qc = QuantumCircuit(1, 1)
qc.measure(0, 0)

# If bit 0 is 1, perform an x-gate, else do nothing
with qc.if_test((0, True)):
    qc.x(0)

qc.draw('mpl')

This can also be written in the older c_if (classical-if) way by doing:

qc = QuantumCircuit(1, 1)
qc.measure(0, 0)
qc.x(0).c_if(0, True) #If bit 0 is True (1) do x-gate on Q0
qc.draw('mpl')

So we see that reset is nothing more than fixing the state of the qubit via measurement, and then flipping the bit if it is found to be in the \(|1\rangle\) state. Reset by itself has limited use outside of qubit initialization at the beginning of a circuit. However, the combination of measurement+reset is a critical component of many dynamic circuits as it allows for qubit measurement and recycling.

BV as a dynamic circuit#

The BV circuit is an example of a circuit that benefits from qubit recycling. To see how this works, let us look at the basic building block of the BV circuit

qc = QuantumCircuit(5, 4)
qc.x(4)
qc.h(4)
qc.barrier()

for i in range(4):
    qc.h(i)
    qc.cx(i, 4)
    qc.h(i)
    qc.measure(i, i)
    qc.barrier()


qc.draw("mpl")

If we could reuse qubits, then the computation of any number of bits would require at most two-qubits. With measure+reset pairs (dynamic circuits) at our disposal, we can thus greatly improve the output fidelity of our BV circuits on hardware.

def dynamic_bv(bitstring):
    """Create a Bernstein-Vazirani circuit from a given bitstring.
    
    Parameters:
        bitstring (str): A bitstring.
        
    Returns:
        QuantumCircuit: Output circuit.
    """
    qc = QuantumCircuit(2, len(bitstring))
    
    # Prepare the |-x> state on target qubit
    qc.x(1)
    qc.h(1)
    
    # For each bit (0 or 1) build a simple circuit block
    for idx, bit in enumerate(bitstring[::-1]):
        
        # Initial H gate on control
        qc.h(0)
        # If bit=1, do a CNOT gate
        if int(bit):
            qc.cx(0, 1)
        # Final H gate to convert phase to computational-basis
        qc.h(0)
        # Measure
        qc.measure(0,idx)
        
        # If not at the final bit, recycle and reset qubits
        if idx != (len(bitstring)-1):
            qc.barrier([0,1])
            # Reset control qubit for reuse
            qc.reset(0)
            # reset target qubit to minimize dephasing
            qc.reset(1)
            # Prepare the |-x> state on target qubit again
            qc.x(1)
            qc.h(1)

    return qc

dyn_qc = dynamic_bv('1'*5)
dyn_qc.draw('mpl')

dyn_ideal_counts = ideal_simulator.run(dyn_qc, shots=10000).result().get_counts()

dyn_ideal_counts

{'11111': 10000}

Now, let us see how our dynamic BV circuits work on hardware:

dyn_bv_circs = [dynamic_bv('1'*N) for N in range(2, MAX_BITS+1)]

dyn_trans_qc = transpile(dyn_bv_circs, backend, optimization_level=3)

dyn_real_counts = backend.run(dyn_trans_qc, shots=10000).result().get_counts()

dyn_fidelities = [hellinger_fidelity(cnts, {'1'*(idx+2): 1}) for idx, cnts in enumerate(dyn_real_counts)]

fig, ax = plt.subplots()
ax.plot(range(2, MAX_BITS+1), noisy_sim_fidelities, 'o--', ms=10, label='Simulation')
ax.plot(range(2, MAX_BITS+1), real_fidelities, 'o--', ms=10, label='Hardware')
ax.plot(range(2, MAX_BITS+1), dyn_fidelities, 'o--', ms=10, label='Dynamic-Hardware')
ax.set_xlabel('Bits')
ax.set_ylabel('Fidelity')
ax.legend();

Dynamic BV: further optimization#

In the dynamic version of BV circuits, there are measure operations followed by a reset operation. As mentioned, earlier, the reset operation in IBM systems is a measure followed by a conditional X gate. The full circuit looks like something like this

qc = QuantumCircuit(1, 1)
qc.measure(0, 0)
qc.reset(0)
qc.draw('mpl')

qc2 = QuantumCircuit(1, 2)
qc2.measure(0, 0)
qc2.measure(0, 1)
qc2.x(0).c_if(1, True)
qc2.draw('mpl')

Now, we have already seen that measurements are the most error prone operations on IBM Quantum systems, and take quite a bit of time (~2 CX gates worth of time). On this device, the average measurement error rate is

np.mean([backend.properties().readout_error(i) for i in range(backend.num_qubits)])

0.015459259259259254

So every measure+reset pair is quite costly in terms of both error and duration. If we could reduce the number of measurements, then the performance would greatly improve. It turns out that we can. The back-to-back measurments in the above case are done immediately, and therefore, a qubit state does not have the scope to evolve between the measurements. In other words, qubit state remains same between those immediate back-to-back measurements.

Thus, instead of back-to-back measurements, we can do only the first measurement and apply the conditional X gate based on that measurement outcome, and in the process, can omit one costly measure operation. (This is an example of the Zeno effect; the state of a system is frozen upon fast repeated measurements.)

Because measure+reset are so ubiquitous in dynamic circuits, Qiskit can auto-simplify combinations of operations. This is done via the ResetAfterMeasureSimplification transpiler pass:

from qiskit.transpiler import PassManager
from qiskit.transpiler.passes.optimization import ResetAfterMeasureSimplification

pm = PassManager(ResetAfterMeasureSimplification())

collapsed_qc = pm.run(qc)
collapsed_qc.draw('mpl')

We are now in a position to further optimize our dynamic BV circuits by collapsing measure+reset pairs down to just a single measurement and conditional X-gate. However, our barrier placement up until now prevents us from optimizing this via Qiskit. As such, we need to define a new set of circuits (instead of having the measure and reset operations on the opposite side of a barrier, the new circuits will have them after the barrier):

def dynamic_bv2(bitstring):
    """Create a Bernstein-Vazirani circuit from a given bitstring.
    
    Parameters:
        bitstring (str): A bitstring.
        
    Returns:
        QuantumCircuit: Output circuit.
    """
    qc = QuantumCircuit(2, len(bitstring))
    
    # Prepare the |-x> state on target qubit
    qc.x(1)
    qc.h(1)
    
    # For each bit (0 or 1) build a simple circuit block
    for idx, bit in enumerate(bitstring[::-1]):
        
        # Initial H gate on control
        qc.h(0)
        # If bit=1, do a CNOT gate
        if int(bit):
            qc.cx(0, 1)
        # Final H gate to convert phase to computational-basis
        qc.h(0)
        # Measure
        qc.barrier([0,1])
        qc.measure(0,idx)
        # If not at the final bit, recycle and reset qubits
        if idx != (len(bitstring)-1):
            # Reset control qubit for reuse
            qc.reset(0)
            # reset target qubit to minimize dephasing
            qc.reset(1)
            #qc.barrier([0,1])
            # Prepare the |-x> state on target qubit again
            qc.x(1)
            qc.h(1)

    return qc

dyn_bv_circs2 = [dynamic_bv2('1'*N) for N in range(2,MAX_BITS+1)]

dyn_bv_circs2[0].draw("mpl")

dyn_trans_qc2 = transpile(dyn_bv_circs2, backend, optimization_level=3)

collapsed_circs = pm.run(dyn_trans_qc2)

collapsed_circs[0].draw('mpl', idle_wires=False)

# Set `dynamic=True` if there is explicit conditional gates
dyn_opt_counts = backend.run(collapsed_circs, shots=10000, dynamic=True).result().get_counts()

dyn_opt_fidelities = [hellinger_fidelity(cnts, {'1'*(idx+2): 1}) 
                        for idx, cnts in enumerate(dyn_opt_counts)]

fig, ax = plt.subplots()
ax.plot(range(2, MAX_BITS+1), noisy_sim_fidelities, 'o--', ms=10, label='Simulation')
ax.plot(range(2, MAX_BITS+1), real_fidelities, 'o--', ms=10, label='Hardware')
ax.plot(range(2, MAX_BITS+1), dyn_fidelities, 'o--', ms=10, label='Dynamic-Hardware')
