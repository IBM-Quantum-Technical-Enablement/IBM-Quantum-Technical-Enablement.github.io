Precision, Accuracy and Number of Shots#

We explore why increasing the number of shots can be beneficial and how the results obtained depend on the Accuracy set by error rates as well as by the Precision set by number of shots.

Load modules, account and backend#

import numpy as np
from qiskit import *
import matplotlib.pyplot as plt
%config InlineBackend.figure_format='retina'
plt.style.use('ibmq-dark')

IBMQ.load_account()
provider = IBMQ.get_provider(group='deployed')

backend=provider.backend.ibm_cairo

Explore maximum number of shots#

On some providers, the maximum number of shots was recently increased to up to 100.000 shots

backend.configuration().max_shots

100000

Alternatively, using the new class of backends BackendV2, max_shots is just an attribute of this class. Access to backends using BackendV2 is managed by qiskit-ibm-provider, which you may install by running

pip install qiskit-ibm-provider

from qiskit_ibm_provider import IBMProvider
provider = IBMProvider()

backend=provider.backend.ibm_cairo
print("version =",backend.version,", max_shots =",backend.max_shots)

version = 2 , max_shots = 100000

Let us explore why setting such a high number of shots has advantages over systems and providers that offer less shots

Set up an exemplary circuit#

N = 4
qc = QuantumCircuit(N)
qc.x(range(N))
qc.h(range(N))
qc.ch(2,1)
qc.ch(1,0)
qc.ch(2,3)
qc.measure_all()
qc.draw('mpl')

Let us see which expectation value this circuit produces. In order to do so, we set up the same circuit without measurements, simulate the resulting Statevector and use the method expval from mthree. In order to get a copy of the circuit qc rather than modify the circuit itself, we set the argument inplace=False .

qc_nomeasure = qc.remove_final_measurements(inplace=False)

from qiskit.quantum_info import Statevector
st = Statevector.from_instruction(qc_nomeasure)
st.probabilities_dict()

# pip install mthree
import mthree
mthree.utils.expval(st.probabilities_dict())

0.4999999999999999

So the ideal expectation value is 0.5, but how close can we get in real experiments?

trans_qc = transpile(qc, backend, optimization_level=3)

trans_qc.draw('mpl',idle_wires=False)

Run on real backend with varying number of shots#

We will run this circuit with a number of shots ranging from 100 up to 10.000

samples = np.array([100, 250, 500, 1000, 2500, 5000, 10000])

The transpiled circuit is run 100 times for each number of shots

jobs = []
for shots in samples:
    jobs.append(backend.run([trans_qc]*100, shots=shots))

fig, ax = plt.subplots()

for idx, shots in enumerate(samples):
    counts = jobs[idx].result().get_counts()
    exp_vals = [mthree.utils.expval(cc) for cc in counts]
    ax.plot([shots]*100, exp_vals, 'o')

ax.axhline(0.5, lw=1.5, linestyle='dashed', color='1', label="Theoretical expectation value")
    
# the standard deviation is proportional to 1/sqrt(N)    
ax.plot(samples, 2/np.sqrt(samples)+np.mean(exp_vals), lw=3, linestyle='dotted', color='white')

ax.plot(samples, -2/np.sqrt(samples)+np.mean(exp_vals),  lw=3, linestyle='dotted', color='white', label="Precision set by number of shots")

ax.set_ylim([0.19,0.71])
ax.set_xscale("log")
ax.axhline(np.mean(exp_vals), lw=1.5, linestyle='dotted', color='grey', label="Mean expectation value")

ax.set_xlabel('Shots')
ax.set_ylabel('Expectation value')
ax.set_title('Precision of repeated executions vs. shots');
ax.legend(loc='lower center');

The Accuracy set by the errors of the involved gates is given by the difference between the mean expectation value vs the ideal value.

Note how the variance of expectation values is reduced with increasing number of shots. The standard deviation is proportional to \(\frac{1}{\sqrt{N}}\), where N is the number of shots as in Shot noise. While you might be lucky and end up close to the ideal expectation value for single runs with a lower number of shots, there is great variance and so it is hard to reproduce the same result each time a circuit is run. In order to reduce this statistical variance, a higher number of shots is recommended, depending on the Precision needed for the experiment in question. Again, the resulting standard deviation can be estimated by \(\frac{1}{\sqrt{N}}\).
